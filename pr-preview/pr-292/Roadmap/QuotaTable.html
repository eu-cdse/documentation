Quotas and Limitations

Copernicus General Users

General Users are all users registered on Copernicus Data Space Ecosystem on the basis of self registration and not falling into the user typologies defined in the subsequent sections. There are no restrictions on who can register to access the data.

# Limitations/Quotas Sentinel Hub APIs⁸ Data Workspace API OData, STAC, S3 Direct HTTP access to COGs openEO API / Algorithm plaza
1 Requests per month 10 000 n/a 50 000⁷ ⁹ n/a
2 Requests per minute 300 n/a 12¹¹ ¹³
3 Processing units (PU) per month 10 000 n/a
4 Processing units (PU) per minute 300
5 Bandwidth limit per connection (MB/s) (IAD¹) n/a 20
6 Number of concurrent connections limit (IAD¹) 4
7 Monthly transfer limit (TB) (IAD¹)¹⁰ 6
8 Number of concurrent orders limit (DAD²) 1 n/a
9 Monthly transfer limit (TB) (DAD²) 0,1
10 Processed Products per month 25
11 Concurrent Processing 2¹² 2
12 Number of active sessions³ 100 n/a
13 A token stays active for⁴ 10 minutes
14 A token can be refreshed in⁵ 60 minutes
15 Number of products that be accessed with one token⁶ No limits
16 Credits per month n/a 1000
17 Concurrent API requests 2


¹ IAD: Immediately Available Data.
² DAD: Deferred Available Data (known as Offline data). It is not possible to order DAD by using OData,STAC or S3, Copernicus Browser or any of the Sentinel Hub APIs. DAD (Offline data) can only be ordered by using the Data Workspace. Only after ordering, it can be downloaded from the catalogue by using the download APIs.
³ This includes, among others, a newly generated token and logging in to the user panel. Please refer to: Keycloak Account Management and see the number of signed in devices - It is not possible to increase this number to a bigger value than 100, with a paid plan. Each session starts when the user generates a new token and use it for the downloading process. One session ends when the token reaches its expiry time (a token stays active for 10 min.
⁴ After reaching this limit, the Access Token must either be refreshed by using the Refresh Token.
⁵ Anytime within 60 minutes after the access token is generated.
⁶ As long as any other limit(s) are not breached.
⁷ Extensions are available via CREODIAS offering (see 3.2.3 Product Generation Service of the Roadmap).
⁸ Similar principles apply for all SentinelHub APIs while the differences and details are covered in the Processing Unit section of our documentation. Note that there are APIs that are not available to Copernicus General Users such as Sentinel Hub Batch Processing API.
⁹ Technical limitation may be applied to maintain platform stability.
¹⁰ After reaching this monthly transfer limit, the maximum bandwidth drops to 1MB/s and the number of concurrent connections drops to 1.
¹¹ Only enabled for openEO synchronous execution requests (`POST /result`) and batch execution requests (`POST /{job_id}/result`).
¹² The maximum number of simultaneous production orders running in parallel, i.e. the orders in “In progress” status.
¹³ 1 request per 5 seconds.

These limitations apply to the Copernicus General user type. Each Copernicus privileged user type (Copernicus Service User, Copernicus International Partner...) may be given other quotas. Please contact Copernicus Data Space Ecosystem Support Team via Submit a request for more information.

The following limitations and quotas are monthy and reset on the 1st of every month:

1 Requests per month
2 Requests per minute
3 Processing units (PU) per month
4 Processing units (PU) per minute
7 Monthly transfer limit (TB) (IAD¹)¹⁰
8 Number of concurrent orders limit (DAD²)
9 Monthly transfer limit (TB) (DAD²)
10 Processed Products per month
16 Credits per month
