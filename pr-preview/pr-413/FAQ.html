Frequently Asked Questions

General

1.What is the phase-out timing for the current Copernicus Data Hub distribution services?

The legacy Copernicus Data Hub distribution service will remain in operations until end of June 2023 to allow a smooth migration to the new Copernicus Data Space Ecosystem by all user communities. The Copernicus Data Hub distribution service will continue offering access to Sentinel data with a gradual ramp-down of the operations capacity and data offering until end of September 2023.

2.In comparison to the existing legacy Copernicus Data Hub, what will be the free services other than stac/cog?

Compared to existing Copernicus Data Hub, there will be additional APIs - OGC interfaces (WMS, WMTS, WCS), OpenEO, Sentinel Hub API, S3, and others. Please refer to the Roadmap for more information on the timing of these interfaces.

3.Is there an end user document that describes specific data products and specific services that are available?

The user level details for every service and dataset will be provided in this documentation. In accordance with each and every every service and dataset embedded into the ecosystem, this documentation will be updated.

4.How long is the project timescale in total?

The time scale of the project is 6 years (i.e. to the end of 2028) with an optional extension up to 4 years (i.e. 2032).

5.Can anyone outside from Europe have free access to any data?

Yes, data and services will be available to users worldwide. You can visit Terms and Conditions for more information.

6.Is there any difference between EU users and non-EU users?

There is no difference between EU users and non-EU users. That said, there will be a continuity of the accounts with higher throughput, managed by ESA (i.e. Copernicus Services, International Partners, etc.).

Data

1.What data will be offered online and what is the timeline for the following months?

For the details on the data offer and timing, please refer to the Roadmap.

2.Is there a page that indicates anomalies with the datasets?

The Copernicus Sentinel Operations Dashboard provides details of events over the past three months that have impact on the completeness of the data production, such as planned calibration activities, manoeuvres, or anomalies. The information of which data is affected is included.

3.With regard to cloud native formats/interfaces, will the data also be available in the original data formats (e.g. for data downloading)?

Yes, data will also be available in original data formats (i.e. .SAFE).

4.Will data, such as Sentinel-2, be processed to a consistent version?

The Sentinel-2 data will be available at the latest processing baseline. And with the reprocessing of Sentinel-2 happening in parallel (out of scope of this project), these will become available on this service as well.

5.Is it possible to download a subset of data corresponding to an AOI, instead of the whole image?

Yes, it is possible to download a subset of data, by using the dedicated APIs, i.e. Sentinel Hub, OpenEO.

6.Are there data offered in the Cloud Optimized Geotiffs (COG) format?

Yes, Sentinel-1 GRD Level 1 data will be available in COG format.

7.When Online data is mentioned, does that mean the data are not on tape?

The Online data or Immediate Access Data (IAD) we are referring to, are indeed not on the tapes. Tapes will still be there for redundancy reasons.

8.Can we download the data acquired by all Sentinel missions (1, 2, 3, 5P, 6) and the other satellites (e.g. Meteosat) via the new interface? Considering some missions are not managed by ESA, but by EUMETSAT for example.

Initially Sentinel 1, 2, 3, 5P and Belgian Collaborative Ground Segment hosted data are planned to be available. The Data Table shows how the Copernicus Data Space Ecosystem will be continously upgraded and how more data will become available.

9.Will it still be the case that data is labelled as “Online/Offline” on the current legacy portal?

Yes, however the vast majority of the data will be Online: all Sentinel-2 L1C/L2A, Sentinel-1 SLC/GRD and just about all other relevant data collections.

10.Will the new interface offer EO ready-to-use products or just L0 and L1 data?

Up to L2 products will be available. The Roadmap shows how the Copernicus Data Space Ecosystem will be continuously upgraded and how more data become available.

11.What is the highest resolution SAR data available in Copernicus Data Space Ecosystem?

The Sentinel-1 SAR achieves a spatial resolution of approximately 5 by 20 m. More info can be found in here.

12.Is it possible to acquire compressed data?

Copernicus Data Space Ecosystem provides access to EO data in zip format without compression.

Services

1.Are the free offering and commercial offering integrated to facilitate the transfer of the users from free to commercial?

Yes, there will be a common user identity, which will allow registered users to seamlessly transfer between two mentioned systems.

2.When we develop an EO ready-to-use product, could we integrate it into the interface and ask the payment from clients?

Yes, commercial services can be built on top, similar to Copernicus Open License.

3.Can users come with wish-list on services?

Yes, users can come up with suggestions to improve or expand the service portfolio. Users can post their improvement suggestions to Community Forum for further evaluation.

4.Is there any limitation on the maximum number of downloads at a time?

Yes, there will be quotas and limitations for different user types. For example, the number of concurrent connections limit for the Copernicus General user type is 4, whereas for the Copernicus Services type, it is 20. For more information, please refer to the Quotas and Limitations section of our documentation.

5.Is it possible to download Sentinel-2 data for a large area at a high resolution in the Copernicus Browser?

Depending on your use, we suggest to use the high-res print (via the high-res print tab) where you will get large areas in a high resolution (the data is though not georeferenced). If you need georeferenced data, split your area in several smaller images that you download or choose a bit lower resolution to stay within the limits of 2500px.

6.Where can I find more information regarding the quotas and limits for accessing data and using the services through your platform?

Please refer to the Quotas and Limitations section of our documentation.

7.How can I request to change my Copernicus user type from Copernicus General user to Copernicus Services user?

Please be aware that Copernicus Services user type is allocated for Copernicus Services and authorised users (e.g. European Union institutions and bodies, National or regional public authorities in the EU or Copernicus Participating States). Submit a Request to our support team. Please select ‘Other’ for the ’I have a question about*:’ and provide the following information: Contract identifier (name, date), Contracting Authority’s contact person, Specific demands (such as Access to CCM data collections, Custom quotas).

Registration and authentication

1.I’m having troubles with registering, what can I do?

Please send an e-mail to help-login@dataspace.copernicus.eu for direct support on this matter.

2.I’m having problems when I try to login or submit a request in the Help Center. How can I solve it?

This may be due to your browser settings. Please make sure Enhanced Tracking Protection is turned off and in the Privacy&Security section of your browser, make sure the option Standard (not Strict) is selected. If these doesn’t solve your issue. please contact our support team via Submit a Request

APIs

1.Where can I find detailed information regarding the duration of Access tokens and Refresh tokens?

The access token is valid for 10 minutes, after 10 minutes, it expires. When Access Token expires, it must either be refreshed by using the Refresh Token or be re-generated. The refresh token is valid for 60 minutes and can be used multiple times within that 60 minutes. The returned access token is again valid for 10 mins. For more information, please refer to the Quotas and Limitations section of our documentation. Also please refer to the OData section of our documentation for more information on Access and Refresh Tokens.

2.Is a STAC catalog planned ? Will the data be accessible on cloud object storage (S3)?

STAC Catalog API is indeed planned. The data is already available over S3 as well.

3.Will Long Term Archive (LTA) process be discontinued when all archived data become Online?

There will still be services available for so called “deferred data access”: data collections that are not commonly used. That said, all of the most relevant collections will be available Online. The Roadmap shows how the Copernicus Data Space Ecosystem will be continously upgraded and how more data become available.

4.Will the platform use STAC standards?

STAC Product Catalog is available. However there may be issues with using the current version with generic STAC libraries. Our dedicated teams are actively working on its development to ensure a seamless experience for all users. Nevertheless it already supports basic product search.

5.Any plan to offer the Pangeo platform for a “pythonist”?

This is currently not in the offer or roadmap.

6.How do I generate S3 access and secret keys?

Please refer to the Access to EO Data via S3 page for the guidance on generating S3 access and secret keys.

7.Which one amongst the 4 catalog APIs (OData, STAC, OpenSearch, Sentinel Hub catalogue) is updated first when new products are published?

OpenSearch, OData and STAC catalog APIs all use the same backend database. Sentinel Hub catalog API contains a subset of the collections, hence it works only for the ones that have been imported to Sentinel Hub. Therefore there is no first updated one.

8.What is the limitation of the number of requests that I can do at the time?

For concurrent requests and other limitations please refer to the Quotas and Limitations section of our documentation for more information.

9.Can I connect directly to the S3 bucket using AWS S3 commands with the S3 keys provided or do I have to use “s3cmd” to download images?

Yes, you can connect to S3 bucket using AWS S3 connection. However some functionality may not be supported. It is recommended to use the ‘s3cmd’ command to download products.

10.Do you have to authenticate for requesting through OpenSearch API?

There’s no need to use any user or authentication when you want to search. User authentication is required for downloading products.

11.Can we use the Sentinel Hub bucket and fetch the products based on the id we obtained from OpenSearch API?

Yes, you can use Sentinel Hub bucket in addition to some programming tools by providing product ID obtained by using OpenSearch API or OData of the Copernicus Dataspace Ecosystem.

12.What is the benefit of fetching imagery from Copernicus Dataspace S3 bucket?

Downloading products via S3 is faster as it delivers products as a .zip archive, skipping the need of zipper.

13.On which region resides the Copernicus Dataspace S3 bucket?

The repo is located in Warsaw/Poland.

14.How can I search for the product in S3 bucket?

Searching via ID or product name in the OpenSearch or OData will give the S3 path to the product in response.

Documentation

1.Which distribution channels will be available for high-throughput data access? Does the public side have user tiers, or is high-throughput data transfer (such as EODATA only a paid service?)

All distribution options (i.e. OData, S3, Sentinel Hub,..) will be constrained with user quotas, which includes both bandwidth limitation, as well as monthly data transfer limits. Please refer to the Quotas and Limitations section of our documentation for more information.

2.Where can I find more information regarding the cost of the “extra” services?

Pricing will be published soon.

3.This December advertisement of the Copernicus Data Space Ecosystem indicates that “For those interested in processing, there will be scalable cloud resources available, optimized for EO tasks”. Does this refer to the current CreoDIAS resources, or something completely new that hasn’t been addressed yet?

Scalable cloud resources will be part of the commercial offering and can be obtained at https://creodias.eu/in first instance. ICT-wise, there will be two options, including Open Telekom Cloud.

4.Are there tutorials (online & physical meetings) to use the new interface?

Tutorials will be added to the documentation in due time explaining the usage of the different interfaces. We will also be present on different conferences explaining the service & ecosystem.
